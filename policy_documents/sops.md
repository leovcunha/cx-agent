# Standard Operating Procedures (SOPs)

## SOP 001: Account Password Reset
**Category:** `account_access`
**Applies To:** All Businesses

**Policy:**
1. If a user requests a password reset, first ask them to confirm the email address associated with their account.
2. If the user provides an email address, inform them that a password reset link has been sent to that address.
3. If the user states they no longer have access to that email address, escalate the ticket to a human agent immediately as this requires manual identity verification.

---

## SOP 002: SaaS Platform Connection Errors
**Category:** `product_troubleshooting`
**Applies To:** SaaS Business Profile

**Policy:**
1. When a user reports that they cannot connect to the platform (e.g., 502 Bad Gateway, timeout), ask them if they are accessing the platform via the Web Portal or the Desktop App.
2. If Desktop App: Instruct the user to clear their local cache by going to Settings -> Clear Cache, and ask them to try again.
3. If Web Portal: Ask the user to verify if they are on a company VPN. If they are, instruct them to disconnect from the VPN and try again.
4. If the issue persists after these steps, ask for their Organization ID and escalate to technical support.

---

## SOP 003: Feature Request Handling
**Category:** `feature_request`
**Applies To:** All Businesses

**Policy:**
1. Thank the user for their suggestion.
2. Ask the user for a brief description of the use-case for this feature (e.g., "Could you share a bit about what problem this feature would solve for you?").
3. Once the user provides the use-case, inform them that the feedback has been logged with the product team and close the conversation.
